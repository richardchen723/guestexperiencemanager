Hostaway Messages API

Third‑party integration reference for the Hostaway Messages dashboard.

Basics

Base URL: use the same host and port as this app.

Authentication: send an API key with each request.

X-API-Key: <your_api_key>
Authorization: Bearer <your_api_key>

Errors return JSON: {"error":"message"}

Example

curl -H "X-API-Key: <your_api_key>" \
  http://127.0.0.1:5001/api/listings

API Key Management (Admin)

Create and revoke API keys used by third parties.

  • GET/admin/api/api-keys — list keys (metadata only).
  • POST/admin/api/api-keys — create key. Body: {"name":"Partner"}.
  • POST/admin/api/api-keys/{api_key_id}/revoke — revoke key.

Health

Use this to verify the service is alive and can reach the database.

  • GET/health — returns status and timestamp.

Listings, Insights, Tags

Listings and AI insights are read‑only. Tags are editable and can be attached to listings.

  • GET/api/listings — list all listings. Query: tags, tag_logic.
  • GET/api/insights/{listing_id} — insights for a listing. Query: refresh=true to re‑run analysis.
  • GET/api/tags — list tags and usage counts.
  • POST/api/tags — create tag. Body: {"name":"pool","color":"#00AAFF"}.
  • DELETE/api/tags/{tag_id} — delete tag.
  • GET/api/tags/autocomplete — suggestions. Query: q.
  • GET/api/listings/{listing_id}/tags — list tags on listing.
  • POST/api/listings/{listing_id}/tags — add tags. Body: {"tags":["dock","pet-friendly"]}.
  • DELETE/api/listings/{listing_id}/tags/{tag_id} — remove tag from listing.

Dashboard

Single endpoint used by the UI to hydrate the dashboard.

  • GET/dashboard/api/data — dashboard cards and summaries. Query: ticket_limit, occupancy_months.

Tickets

Full read/write ticketing API. Supports filters, tags, comments, images, and recurring tasks.

  • GET/tickets/api/tickets — list tickets. Query: listing_id, listing_ids, assigned_user_id, status, priority, category, issue_title, tags, tag_logic, search, past_due, recurring, due_days.
  • GET/tickets/api/tickets/{ticket_id} — ticket with comments.
  • POST/tickets/api/tickets — create ticket. Body includes title, issue_title, optional listing_id or listing_ids.
  • PUT/tickets/api/tickets/{ticket_id} — update ticket fields.
  • DELETE/tickets/api/tickets/{ticket_id} — delete ticket (admin only).
  • GET/tickets/api/tickets/{ticket_id}/comments — list comments.
  • POST/tickets/api/tickets/{ticket_id}/comments — add comment. Body: {"comment_text":"..."}.
  • DELETE/tickets/api/tickets/{ticket_id}/comments/{comment_id} — delete comment.
  • POST/tickets/api/tickets/suggest — AI suggestion. Body: {"listing_id":123,"issue_title":"...","issue_details":"..."}.
  • GET/tickets/api/listings/{listing_id}/issues — list issues from insights.
  • GET/tickets/api/users — list approved users for assignment.
  • GET/tickets/api/tickets/{ticket_id}/tags — list tags on ticket.
  • POST/tickets/api/tickets/{ticket_id}/tags — add tags. Body: {"tags":["hvac","vendor"]}.
  • DELETE/tickets/api/tickets/{ticket_id}/tags/{tag_id} — remove tag from ticket.

Images

Upload with multipart/form-data field name image.

  • POST/tickets/api/tickets/{ticket_id}/images — upload ticket image.
  • GET/tickets/api/tickets/{ticket_id}/images — list ticket images.
  • GET/tickets/api/tickets/{ticket_id}/images/{image_id} — fetch ticket image file (scoped).
  • GET/tickets/api/tickets/{ticket_id}/images/{image_id}/thumbnail — fetch ticket thumbnail (scoped).
  • DELETE/tickets/api/tickets/{ticket_id}/images/{image_id} — delete ticket image.
  • POST/tickets/api/comments/{comment_id}/images — upload comment image.
  • GET/tickets/api/comments/{comment_id}/images — list comment images.
  • GET/tickets/api/comments/{comment_id}/images/{image_id} — fetch comment image file (scoped).
  • GET/tickets/api/comments/{comment_id}/images/{image_id}/thumbnail — fetch comment thumbnail (scoped).
  • DELETE/tickets/api/comments/{comment_id}/images/{image_id} — delete comment image.
  • GET/tickets/api/images/{image_id} — fetch image file.
  • GET/tickets/api/images/{image_id}/thumbnail — fetch thumbnail.
  • POST/tickets/api/recurring/process — process recurring tasks (admin only).

Reviews

Review filters are scoped to the current user. You can list unresponded reviews and saved filters.

  • GET/reviews/api/unresponded — unresponded reviews. Query: tag_ids.
  • GET/reviews/api/filters — list saved filters.
  • POST/reviews/api/filters — create filter. Body: {"name":"Low ratings","tag_ids":[1,2],"max_rating":4,"months_back":3}.
  • PUT/reviews/api/filters/{filter_id} — update filter.
  • DELETE/reviews/api/filters/{filter_id} — delete filter.
  • GET/reviews/api/filters/{filter_id}/reviews — get reviews. Query: sort_by, sort_order.

Knowledge Base

Upload and manage documents, attach listings and tags, and perform search.

  • POST/knowledge/api/documents — upload document. Multipart fields: document, title, listing_ids, tag_names, is_admin_only.
  • GET/knowledge/api/documents — list documents. Query: listing_id, listing_ids, tag_ids, search, page, per_page.
  • GET/knowledge/api/documents/{document_id} — document metadata.
  • GET/knowledge/api/documents/{document_id}/file — download or view. Query: download=true.
  • POST/knowledge/api/documents/search — search. Body: {"query":"heater reset","listing_ids":[1],"tag_ids":[3],"limit":50}.
  • PUT/knowledge/api/documents/{document_id} — update metadata.
  • DELETE/knowledge/api/documents/{document_id} — delete document.

Sync

Trigger and monitor sync runs.

  • GET/sync/api/history — list recent sync runs.
  • GET/sync/api/running-status — running syncs.
  • GET/sync/api/job/{job_id}/detail — job detail.
  • GET/sync/api/{sync_run_id}/detail — run detail.
  • POST/sync/api/full — trigger full sync.
  • POST/sync/api/incremental — trigger incremental sync.
  • GET/sync/api/status/{job_id} — status by job id.

Users and Auth

User profile and admin user management APIs.

  • GET/auth/api/profile — current user profile.
  • PUT/auth/api/profile — update profile fields.
  • GET/admin/api/users — list users (admin only).
  • POST/admin/api/users/{user_id}/approve — approve user.
  • POST/admin/api/users/{user_id}/revoke — revoke user.
  • POST/admin/api/users/{user_id}/role — update role.
  • DELETE/admin/api/users/{user_id} — delete user.

Activities

Admin reporting and audit trails.

  • GET/admin/api/activities — query activity logs.
  • GET/admin/api/activities/reports/ticket-metrics — ticket metrics report.
